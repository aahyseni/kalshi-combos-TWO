Honest findings (not all of them flatter us)

ML×2 parlays carry an 18.3¢ median gap (n=485) — our independence fair sits far above maker clearings on 2-leg cross-game moneyline parlays: the winner's-curse tail, concentrated. Identical under both configs (not a pair-table issue) — this is markup/quoting-policy territory, echoing the WC finding. Note the pattern by legs: ML×2 18.3¢ → ML×4 0.80¢ → ML×6 0.29¢ — deeper parlays have tiny fairs, so cent errors shrink while relative errors do not.
The mlb_runs structural grid has its own 5.53¢ median error, +4.21¢ bias (n=232, identical both configs — it doesn't read the pair table). 5× the copula's game-line error; flagged as a separate calibration follow-up.
In-play-only combos excluded from the verdict: 5,024 (every print landed after estimated first pitch; cutoff = expiry − 4h, a documented estimate). The recorder's combo-trades poller saw only ~2.5k of an extrapolated ~84k traded MLB combos — clearings are tape-sourced (complete) for exactly this reason.

Method

gather: MLB-strict combos + marginal snapshots from the recorder DB (read-only, bounded chunks) · clearings from Kalshi's authed trade tape (poller-gap-immune) · per-leg settlement/expiry from the public API · strictly-pregame filter keeps prints before min-over-legs(expected_expiration − 4h). price: blind driver over the LIVE pricing modules + shipped config; the legacy config is a runtime deep-copy override (engine untouched, CLAUDE.md rule 8); fairs computed only for tape-printed combos via a prices-free ticker list (selection cannot bias a fair). analyze: this page. Reports channel: docs/reports/.
